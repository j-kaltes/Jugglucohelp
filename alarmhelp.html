Alarms

Glucose alarms:

The stream values received via Bluetooth can be used to set low and high glucose alarms.

Low glucose alarm:

Value below which an alarm should go off.

High glucose alarm:

Value above which an alarm should go off.

Loss of signal alarm:

Specify after how many minutes an alarm should go off if no glucose value is received via Bluetooth.

Alarms will stop when one of the following conditions is satisfied:

- The specified duration has elapsed;

- The curve view is touched;

- The Android notification of Juggluco is touched (in unlocked state);

- Cancel button is touched of the alarm notification in Juggluco.

For each alarm you can specify which Ringtone to play and for how long.

Alarm is alarm:

When set, is the ringtone played for low, high and loss of signal alarms given the Android Alarms category. This means that in Android settings it's volume is managed with the Alarms volume level and it is not silenced when sound of the whole phone is turned off or set to vibration. When Alarm is alarm is not set, the ringtone volume level is used. This is always the case for the value available notification.
